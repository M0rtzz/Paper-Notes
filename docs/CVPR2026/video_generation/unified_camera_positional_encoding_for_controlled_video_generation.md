# Unified Camera Positional Encoding for Controlled Video Generation

**会议**: CVPR 2026  
**arXiv**: [2512.07237](https://arxiv.org/abs/2512.07237)  
**代码**: [https://github.com/chengzhag/UCPE](https://github.com/chengzhag/UCPE)  
**领域**: 视频生成  
**关键词**: 相机位置编码, 视频生成, 镜头畸变, 扩散Transformer, 相机可控生成

## 一句话总结

提出统一相机位置编码 UCPE，将完整的相机几何信息（6-DoF 位姿、内参、镜头畸变）通过相对光线编码和绝对朝向编码注入 Transformer 注意力机制，仅增加不到 1% 可训练参数即实现跨异构相机模型的精细视频生成控制。

## 研究背景与动机

Transformer 已成为 3D 感知、视频生成和世界模型的通用骨干，理解相机几何对于将视觉观测锚定到三维空间至关重要。然而，现有相机编码方法存在严重局限：

现有方法分为两类：(1) **绝对编码**（如 Plücker 编码）将每条光线用方向+力矩 6D 向量表示，但依赖预定义世界坐标系，跨场景泛化性差；(2) **相对编码**（如 CaPE、GTA、PRoPE）通过在注意力中编码相对相机变换来实现坐标系无关性，但**假设针孔投影**，无法处理鱼眼、全景等非线性镜头畸变。

实际应用中（自动驾驶、机器人感知），广角镜头、鱼眼镜头和全景相机普遍使用，强畸变使得基于针孔假设的方法失效。此外，现有文本到视频方法仅定义相对于首帧的相机位姿，留下全局旋转（特别是 pitch 和 roll）的歧义——无法指定或复现初始视角的绝对朝向。

核心洞察：**将编码从"相机级别"细化到"光线级别"**——每个图像 token 对应一条独立的观测光线，在光线空间而非相机空间做相对变换，天然兼容任意投影模型。

## 方法详解

### 整体框架

UCPE 想解决的是一个很具体的痛点：让视频生成模型听懂"相机怎么动、镜头是什么样"的指令，而且这个指令要能跨针孔、鱼眼、全景这些差异巨大的相机模型通用。它的做法是把相机几何拆成两路注入预训练的扩散 Transformer（Wan 模型）——一路管"相对几何"，让每个图像 token 都对应一条独立的观测光线，在光线空间里做相对变换；另一路管"绝对朝向"，用一张重力对齐的方向图锚定 pitch/roll。两路信息都不直接改原网络，而是通过一个轻量适配器旁挂进去，从而保住预训练学到的画面先验。

### 关键设计

**1. 相对光线编码：把"相机级"变换细化到"光线级"，从而兼容任意镜头畸变**

现有相对编码（CaPE、PRoPE 等）都在相机级别操作——同一视图里所有 token 共享一个变换矩阵，这等于默认相机是针孔投影、整幅图遵循同一个线性映射；一旦遇到鱼眼、全景这种空间变化的非线性投影就失效。UCPE 的破局点是把相机统一抽象成一个光线映射函数 $\Phi_\psi: (u,v) \mapsto (o, d)$，每个像素 token 都拿到自己那条光线的原点 $o$ 和方向 $d$，于是畸变信息天然被编码进了每条光线本身，不再依赖统一的投影公式。

具体地，对每个 token $t$ 都建一个局部光线坐标系：以光线方向 $d_t$ 为 z 轴，借相机的下方向定出 x、y 轴，凑成正交基 $R_t^{wr}$，再配上平移得到光线到世界的变换 $T_t^{wr}$。它的逆 $T_t^{rw}$ 当作几何算子，通过 Kronecker 积扩展到特征维度，在注意力里对 query-key 做相对变换：

$$Q_t^\top D_t D_{t'}^{-1} K_{t'}$$

这样注意力捕获的不再是"两个相机之间"的关系，而是"两条光线之间"的几何关系。每个 token 有独立算子，鱼眼把直线弯成弧、全景把一圈视野铺平——这些以前会破坏针孔假设的情况，现在只是不同的光线方向而已，方法照单全收。

**2. 绝对朝向编码（Lat-Up Map）：补上相对编码缺失的"哪边是上"**

相对编码只关心光线之间的相对关系，天生没有绝对参考；而文生视频通常只定义相对于首帧的位姿，于是首帧本身的 pitch/roll 完全是歧义的——你没法指定"初始视角是平视还是俯视"。UCPE 用一张纬度-上方向图把重力方向钉死。纬度图编码每条光线相对水平面的仰角

$$\text{Lat}_t = \arctan2\big(-d_{t,y},\ \sqrt{d_{t,x}^2 + d_{t,z}^2}\big)$$

上方向图则把光线绕局部轴转一个微小角度 $\delta$、投影回图像平面，取归一化的位移方向，刻画"图像里的上在世界里指哪"。两者拼成 3D 向量，经线性投影作为偏置加到 token 特征上。因为它锚的是重力对齐的"上"，模型顺带得到了天空/地面分离这类外观线索，对广角镜头还附带了一点畸变感知能力。

**3. 空间注意力适配器：以 <1% 参数旁挂进预训练模型，不动原先验**

直接替换 Transformer 原有的位置编码会把预训练学到的视觉先验一并破坏掉。UCPE 走 LoRA 风格的旁路：每个 DiT 块里原始自注意力原封不动，再并联一条 UCPE 条件注意力分支。这条分支先用 $1/C$ 维度的降维投影压低参数和算力，输出再经一个零初始化的线性层映射回原维度，与原注意力输出相加。零初始化是关键——训练起点时这条分支输出全零，整个模型行为和预训练模型完全一致，相机感知能力是从零"长"出来的，既避免了灾难性遗忘，又把额外开销压到不足 1% 参数。

### 损失函数 / 训练策略

使用标准扩散损失训练适配器参数。为支持多样化相机训练和评估，构建了覆盖多种内参、畸变配置和运动轨迹的大规模视频数据集。编码中将相对光线编码与 RoPE 混合使用（各占特征维度一半），兼顾光线空间和图像空间推理。

| 配置 | RotErr | TransErr | 说明 |

### 关键发现

- 从相机级到光线级的编码升级在针孔场景下也有提升，说明光线级细粒度几何信息有普遍价值
- Lat-Up Map 不仅解决朝向歧义，还对广角镜头有辅助畸变感知效果
- 适配器仅需 <1% 额外参数，证明几何先验可以极其高效地注入预训练模型
- 在鱼眼/UCM 等非针孔相机上，UCPE 是唯一能正确复现目标畸变的方法

## 亮点与洞察

- **"光线级位置编码"是优雅的统一抽象**：任何相机模型（针孔、鱼眼、全景）都可以抽象为像素到光线的映射，在光线空间做相对变换天然兼容所有投影类型，避免了针对不同相机模型的特殊处理
- **零初始化适配器的设计巧思**：通过零初始化输出投影确保训练起点等价于原始模型，避免了微调大模型时常见的"灾难性遗忘"
- **方法的通用性超越视频生成**：UCPE 可直接用于多视图重建、新视角合成等其他需要相机几何推理的 Transformer 任务

## 局限与展望

- 当前仅在视频生成任务上验证，多视图重建和新视角合成的实验待补充
- 非中心相机（如折反射系统）的光线原点不同，虽然理论上可处理但未实验验证
- 大规模训练数据集的多样性仍依赖合成数据，真实世界异构相机数据稀缺
- Lat-Up Map 假设存在重力方向，对太空等无重力场景不适用

## 相关工作与启发

- **vs Plücker 编码**: 绝对编码，依赖世界坐标系，位姿泛化差；UCPE 的相对光线编码消除了这一依赖
- **vs PRoPE**: 在投影空间做相对编码，但内参矩阵 K 仅适用于针孔模型；UCPE 用光线映射替代投影矩阵
- **vs ReCamMaster/CameraCtrl**: 基于 Plücker 或参数注入的方法，无法处理非线性畸变
